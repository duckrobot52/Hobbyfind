# 정보 구조 정의서 (Information Architecture Document): HobbyFind

본 문서는 Next.js 프레임워크 기반으로 구현되는 취미 큐레이션 웹서비스 **HobbyFind**의 정보 구조(IA)를 UX 설계 및 개발 실습에 즉시 반영할 수 있도록 구체적으로 기술합니다.

---

## 1. 전체 사이트맵 구조 (Site Map)

웹사이트의 전체적인 메뉴 계층 구조와 비로그인 상태에서의 접근 권한을 명시합니다.

```
HobbyFind (Root)
├── 1. 홈 (/) `[비회원 가능]`
├── 2. 카테고리별 페이지 (/category/[type]) `[비회원 가능]`
│   ├── 운동형 (/category/sports)
│   ├── 지능형 (/category/intelligence)
│   └── 예술형 (/category/art)
├── 3. 로그인 (/login) `[비회원 전용]`
├── 4. 회원가입 (/signup) `[비회원 전용]`
└── 5. 마이페이지 (/mypage) `[로그인 회원 전용]`

```

---

## 2. 사용자 흐름 (User Flow)

유저의 인증 상태(비회원 / 회원)에 따른 핵심 이용 시나리오 경로입니다.

### 2.1. 비회원 사용자 플로우

```
[사이트 접속] ──> [홈 (/) 탐색] ──┬──> [카테고리 메뉴 클릭] ──> [카테고리 페이지 이동]
                                  └──> [취미 카드 북마크 클릭] ──> [로그인 (/login) 이동 권유]

```

### 2.2. 회원 사용자 플로우

```
[로그인 완료] ──> [홈 (/) 또는 카테고리 페이지 탐색] ──> [취미 카드 북마크 클릭 (토글 On)] 
                                                                 │
[마이페이지 (/mypage) 이동] <── [통계 차트 및 북마크 목록 확인] <──────┘

```

## 3. 내비게이션 구조 (Navigation Structure)

서비스의 최상단에 고정되는 상단 바(Top Bar)와 하단 바(Footer)의 상태별 노출 제어 방식입니다.

### 3.1. 상단 바 (Top Bar) 상태별 제어

| 구분 | 비로그인 (비회원 상태) | 로그인 (회원 상태) |
| --- | --- | --- |
| **좌측 영역** | • 서비스 로고 (`HobbyFind` 홈 이동) | • 서비스 로고 (`HobbyFind` 홈 이동) |
| **중앙 영역** | • 카테고리 메뉴 (운동형, 지능형, 예술형) | • 카테고리 메뉴 (운동형, 지능형, 예술형) |
| **우측 영역** | • `로그인` 링크 버튼<br>

<br>• `회원가입` 링크 버튼 | • `마이페이지` 링크 버튼<br>

<br>• `로그아웃` 버튼 |

### 3.2. 하단 바 (Footer) 포함 여부 및 요소

* **포함 여부**: 레이아웃 안정성과 접근성 준수를 위해 고정 배치합니다.
* **포함 요소**: 카피라이트 표기(`© 2026 HobbyFind. All rights reserved.`), 이용약관 및 개인정보처리방침 고지 링크(기본 링크).

## 4. 페이지 계층 구조 & 주요 콘텐츠 구성

각 페이지에 포함되는 UI 섹션 구조와 핵심 콘텐츠 요소의 명세서입니다.

### 4.1. 홈 페이지 (`/`)

* **Hero 섹션**: 사이트 핵심 가치를 전달하는 소개 문구 메인 타이틀 배너
* **카테고리 필터 스위치 영역**: 운동형, 지능형, 예술형 3개 분류를 실시간으로 켜고 끌 수 있는 토글 탭
* **메인 콘텐츠 영역**: 18개 고정 취미 목록 전체를 보여주는 썸네일 카드 그리드 구조

### 4.2. 카테고리별 페이지 (`/category/[type]`)

* **카테고리 헤더 섹션**: 현재 선택된 대분류 타이틀 및 각 분류의 특징을 서술한 간단한 요약 설명문
* **필터 콘텐츠 영역**: 해당 대분류(type)에 종속된 6개의 고정 취미 카드를 노출하는 카드 그리드

### 4.3. 회원 관리 페이지 (`/login`, `/signup`)

* **인증 폼 섹션**: 아이디(Text `input`), 비밀번호(Password `input`) 입력 필드와 확인 버튼
* **컨텍스트 피드백 영역**: 잘못된 계정 입력 시 노출되는 실시간 텍스트 에러 메시지창
* **화면 전환 앵커**: 로그인 화면 하단의 '회원가입하러 가기' 링크, 회원가입 화면 하단의 '로그인하러 가기' 링크

### 4.4. 마이페이지 (`/mypage`)

* **대시보드 통계 섹션**: 회원이 저장한 총 북마크 개수 요약 카운터 및 대분류 3종의 분포를 보여주는 원형/도넛 형태의 시각적 통계 차트
* **저장 목록 섹션**: 유저가 북마크 버튼을 활성화해 둔 취미 카드들만 선별하여 수집한 카드 리스트 그리드

## 5. 콘텐츠 조직화 (Content Organization)

화면 내 렌더링되는 핵심 오브젝트의 세부 정보 요소 매핑 테이블입니다.

| 페이지 / 오브젝트 | 콘텐츠 요소 | 데이터 속성 및 UI 노출 형태 |
| --- | --- | --- |
| **공통 취미 카드**<br>

<br>(Hobby Card) | 이미지/썸네일 | • 각 취미의 아이덴티티를 나타내는 시각적 썸네일 자산 |
|  | 취미명 | • 18개 고정 한글 명칭 (`h3` 타이틀 레이블) |
|  | 대분류 태그 | • 운동형, 지능형, 예술형을 구별하는 텍스트 뱃지 UI |
|  | 북마크 토글 버튼 | • 회원이 터치 시 활성화/비활성화 상태를 보여주는 하트/별 모양 아이콘 |
| **마이페이지 통계**<br>

<br>(Stats Section) | 요약 카운터 | • `총 N개의 취미를 보관 중입니다.` (정수 데이터 출력) |
|  | 분포도 그래프 | • 운동형 N%, 지능형 N%, 예술형 N% 차트 컴포넌트 데이터 바인딩 |

## 6. URL 구조 (URL Structure)

Next.js App Router 아키텍처 규칙을 적용한 시맨틱 구조입니다.

* **홈**: `/`
* **로그인**: `/login`
* **회원가입**: `/signup`
* **마이페이지**: `/mypage`
* **카테고리별 페이지**: `/category/[type]`
* 운동형 카테고리 경로: `/category/sports`
* 지능형 카테고리 경로: `/category/intelligence`
* 예술형 카테고리 경로: `/category/art`

## 7. 상호작용 패턴 (Interaction Patterns)

* **카테고리 필터 토글**: 홈 화면에서 특정 카테고리 필터 버튼을 클릭하면, 페이지가 새로고침되지 않고 클라이언트 사이드에서 해당 속성을 가진 취미 카드 리스트만 필터링되어 즉시 페이드인 연출과 함께 정렬됩니다.
* **북마크 토글 동작**:
* 회원이 로그인한 상태에서 버튼 클릭 시 버튼의 색상이 활성화(예: 브랜드 컬러 채우기)되며 데이터베이스에 기록됩니다. 재클릭 시 해제됩니다.
* 비회원이 클릭하면 처리가 중단되고 "로그인이 필요한 기능입니다. 로그인 페이지로 이동하시겠습니까?" 팝업 모달이 노출되며, 승인 시 `/login` 경로로 전환됩니다.


* **로그인 오류 피드백**: 로그인 실패 시 입력 폼 전체가 새로고침되는 현상을 막고, 폼 하단 영역에 붉은색 텍스트로 "아이디 또는 비밀번호가 일치하지 않습니다."라는 명확한 한국어 에러 안내 문구를 동적으로 출력합니다.

## 8. 컴포넌트 계층 구조 (Component Hierarchy)

Next.js 환경에서의 UI 재사용성과 원자 구조 설계를 반영한 컴포넌트 아키텍처 구조도입니다.

```
RootLayout (전역 레이아웃)
├── Topbar (상단 바 내비게이션 컴포넌트)
│   └── CategoryMenu (대분류 이동 링크 그룹)
├── [Main Page Component] (각 라우터별 페이지 진입)
│   ├── Home / CategoryPage
│   │   └── HobbyCardGrid (취미 카드 리스트 그리드 컨테이너)
│   │       └── HobbyCard (개별 취미 카드 오브젝트)
│   │           └── BookmarkButton (북마크 활성/비활성 토글 스위치)
│   ├── Login / Signup Form
│   │   └── ErrorMessage (인증 실패 텍스트 출력 컴포넌트)
│   └── MyPage
│       ├── StatsChart (카테고리 비율 분포 시각화 차트 컴포넌트)
│       └── HobbyCardGrid (내가 저장한 취미 리스트 재사용)
└── Footer (하단 약관 및 저작권 정보 바)

```
